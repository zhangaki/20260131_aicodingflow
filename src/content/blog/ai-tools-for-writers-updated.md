# Publish-Ready SEO Checklist (Pass/Fail)

Use this before you hit publish. Every item is **testable**: it either passes or fails.

If you want a faster, tool-assisted version, run the same checks on our internal page: [SEO Audit](/tools/seo-audit).

## 1) Search Intent Fit

- **PASS** if the first 100 words clearly answer: “Who is this for, and what problem does it solve?”
- **PASS** if the page promises one primary outcome (e.g., “pick an AI writing tool”) and sticks to it.
- **FAIL** if the intro is generic, or the post covers multiple unrelated intents.

Quick fix: rewrite the intro to include audience + outcome + timeframe.

## 2) Title + Meta Description

- **PASS** if the title contains the primary keyword once, naturally.
- **PASS** if the meta description is 140–160 characters and states a benefit + audience.
- **FAIL** if the title is clickbait-y or the meta description is missing/too long.

Quick fix: make the title specific (tool type + outcome) and rewrite the description as “For [audience], get [benefit] without [pain].”

## 3) Headings & “Skimmability”

- **PASS** if there is exactly one `#` (H1) and it matches the topic.
- **PASS** if every section uses `##` (H2) and answers one clear question.
- **PASS** if at least 1 list/table exists that a reader can skim in 10 seconds.
- **FAIL** if headings are vague (“Overview”, “Thoughts”) or you have multiple H1s.

Quick fix: rename H2s to match user questions (e.g., “Which AI tool is best for rewriting?”).

## 4) Tool Selection Is Executable

- **PASS** if the reader can pick a tool in under 2 minutes (clear “best for” and “not for”).
- **PASS** if at least 2 tools are suggested for different jobs (drafting vs editing vs summarizing).
- **FAIL** if the post is only descriptive with no decision rule.

Quick fix: add a 3-row decision table: Job → Best tool type → What to test.

## 5) Workflow Test (10 minutes)

Run this exact mini-test on your top 2 tools.

- **PASS** if you can produce a 150–250 word draft in 10 minutes.
- **PASS** if the output includes at least 3 concrete points (examples, steps, or claims) you can verify.
- **FAIL** if the output is generic filler, repeats itself, or ignores your audience/tone.

Test prompt (copy/paste):

"Write a 200-word intro for: [TOPIC]. Audience: [WHO]. Tone: [TONE]. Outcome: [WHAT THEY WANT]. Include: (1) one clear promise, (2) three bullet points, (3) one next step." 

Quick fix: constrain harder (add structure, length, and 3 required bullets).

## 6) Internal Links (Minimum 2, Relevant)

- **PASS** if you include **2+** internal links to closely related pages.
- **PASS** if anchor text describes the destination (not “click here”).
- **FAIL** if you have zero internal links, or they’re unrelated.

## 7) Final “Publish Gate” (Hard Stop)

- **PASS** if you can summarize the post in 1 sentence: “For [audience], this helps you [outcome] by [mechanism].”
- **PASS** if the page has one primary CTA (e.g., “test 2 tools with the same prompt”).
- **FAIL** if you can’t state the outcome clearly, or there are multiple competing CTAs.

If you fail any item above, don’t publish yet—fix the lowest-effort fail first (usually: intro, title/description, or internal links).
